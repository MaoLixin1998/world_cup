# 新增项目内变更沉淀规则

## 背景

开发过程中只看聊天记录或 git commit，不适合长期学习和复盘。之后每一次改动都需要在对应项目里留下文件，说明为什么改、改了什么、怎么验证。

## 改了什么

- 新增 `docs/development/change-record-rule.md`，定义变更记录放置位置、命名和模板。
- 更新 `README.md`，把变更沉淀规则放到项目入口文档里。
- 新增本次规则变更记录文件，作为之后执行该规则的样例。

## 学习笔记

项目文档可以分两类：

- 规范文件：长期有效，告诉团队以后怎么做。
- 变更记录：针对某一次改动，告诉后来的人当时为什么这么做。

这次 `docs/development/change-record-rule.md` 是规范文件；当前文件是这次改动本身的记录。

## 验证

- 文档改动，无需运行编译或单元测试。
- 已检查当前工作区，避免提交本地遗留的 Gradle wrapper 改动。

## 后续

以后所有代码、配置、UI、架构和文档改动，都要按该规则在对应项目中新增变更记录。
